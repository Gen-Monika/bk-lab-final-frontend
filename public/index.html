Lab 4 · BKVision

用户行为采集面板

Django 中间件采集 CMDB 和 JOB 相关接口访问，将 API 类别、名称、用户、状态码和耗时写入数据库。

  • 统计卡片展示总调用量、API 数和备份动作数。
  • 采集异常被隔离，不影响原业务请求。

事件管理

行为记录归档与多选

本地行为表支持归档、恢复、按归档状态批量选择和反向选择，使大量测试事件也能保持可读。

  • 聚合 API 数据和明细事件都支持整理。
  • 与 Lab3 执行记录管理交互保持一致。

数据源

BKVision 连接 SaaS 数据库

BKVision 通过 SaaS MySQL 数据源读取中间件写入的统计表和明细表，为图表配置提供数据基础。

  • 数据源与应用数据库保持一致。
  • 图表查询使用 SQL 聚合行为数据。

可视化

行为类型分布与接口排行

仪表盘使用饼图、柱状图和明细表展示 API 行为类型分布、接口调用排行和 JOB 操作情况。

  • 选择单人测试也能形成图表的数据维度。
  • 可从最近事件继续追踪用户操作路径。

嵌入展示

SaaS 页面嵌入 BKVision

发布后的仪表盘通过环境变量配置到页面中，以 iframe 方式嵌入，源码中不固定平台生成的私有链接。

  • 保留 JOB Console 和 Host Manager 入口。
  • 仪表盘发布后可在同一 SaaS 页面查看。
